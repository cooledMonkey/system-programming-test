# Контрольная работа по дисциплине "Системное программирование"
# Задание 1
## Описание задачи
Написать многопоточный и асинхронный серверы, которые обрабатывают внешние запросы по страницам сайта https://dental-first.ru/catalog. Для тестирования напишите сервер, который отправляет множественные запросы по различным страницам сайта.
Сами сервера должны сохранять названия товаров в файл. Сравните время выполнения, потребляемую память, и нагрузку на сервер.

**Дополнительно**: добавить подсчет суммарной стоимости всех пропарсенных товаров. Найдите оптимальный вариант решения этой задачи для асинхронного кода.

## Решенние
- Код решения находится в папке exercise_1, многопоточный и асинхронный серверы находятся в файлах с соответствующими названиями.
- В файле client есть возможность проверить работу серверов, чтобы выбрать тип сервера необходимо для переменной server_type указать значение "async" или "threading".
- В списке links содержатся ссылки по которым будут производится запросы.
- Ответом сервера на запрос является сумма стоимостей всех товаров со страницы.
- Названия товаров сохраняются в 2 файла, data_async.txt - для асинхронного сервера, data_threading.txt - для многопоточного.

# Задание 2
## Описание задачи
Напишите асинхронный и многопоточный сокеты, и клиент для них, которые считывают количество строк в файлах некой директории. Сравните показатели потребления, и время выполнения. Опредените, какая минимальная память гарантированно обеспечит обработку 1000 одновременный запросов.

## Решенние
- Код решения находится в папке exercise_2, многопоточный и асинхронный серверы находятся в файлах с соответствующими названиями.
- В файле client есть возможность проверить работу серверов, чтобы выбрать тип сервера необходимо для переменной server_type указать значение "async" или "threading".
- В списке dir_names содержатся пути к папкам в которых будет производится подчсет строк в файлах, необходимо подставить свои пути для анализа чтобы избежать ошибок выполнения программы.
  
Оценка требуемой памяти для такой задачи непростая задача, ведь во многом ответ зависит от количества и размера файлов в заданной директории, поэтому решим опираться на средние значения - 0.3Мб на 1 директорию для асинхронного сервера.
На 100 директрий было затрачено 1.3Мб, на 500 - 2.5Мб, значит можно сделать вывод что на 1000 запросов будет потрачено около 5Мб. Для многопоточного сервера цифры оказались такие: 0.4Мб для одной директории, 2.4Мб для 100 директорий, следовательно можно сделать вывод что на 1000 запросов будет затрачено около 15Мб.
